Programming Tricks in Python3 - Part 2

Hello World! This is my second post on Python Programming tricks and these are actually the snippets which I loved and read on the internet and a book named Python Cookbook. I would like to share some of the concepts/tricks of python which blew my mind , and I actually did them on my repl:

1. defaultdict vs setdefault :

when you want to use dictionary and for every key the datatype of the value is same,so one can use one of the two methods in Python3 namely defaultdict and setdefault

>>>from collections import defaultdict

>>>d = defaultdict(list)
>>>d['a'].append(1)
>>>d['a'].append(2)
>>>d
defaultdict(<class 'list'>, {'a': [1, 2]})

OR with setdefault

>>>d = {}
>>>d.setdefault('a', []).append(1)
>>>d.setdefault('a', []).append(2)
>>>d.setdefault('b', []).append(4)

It would make sense that defaultdict is faster that dict.setdefault() since the former sets its default for the entire dict at creation time, whereas setdefault() does it per element when it is read. One reason to use setdefault is when the default you assign is based on the key (or something) rather than a generic default for the entire dict.

2. Finding Commonalities in Two Dictionaries

You have two dictionaries and want to find out what they might have in common (same keys, same values, etc.).

>>> a={'a' :1, 'b':2,'c':2}
>>> b={'d':1,'e':1,'b':2}

#Common keys
>>> a.keys() & b.keys()
{'b'}


#keys which are in a but not in b
>>> a.keys() - b.keys()
{'c', 'a'}

#(key,value) pairs in common
>>> a.items() & b.items()
{('b', 2)}

3. Naming a slice

This isn't a trick but I found it amazing and important because it can be used for improving the readability of the code. Instead of hardcoding the slices we can name it and change the value of the slice. For example:


>>> a=slice(3,5,1)
>>> b=slice(3,5)
>>> a.start
3
>>> a.stop
5
>>> a.step
1
#by default step is None
>>> b.step
>>> c=[1,2,3,4,5,6,7,8,9]
# it's as good as
>>> c=[1,2,3,4,5,6,7,8,9]
>>> c[b]
[4, 5]
>>> c[3:5]
[4, 5]
