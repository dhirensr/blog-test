Text Retrieval (TR) used by search engines

Text Retrieval (TR)

I'm starting blogging and this is my first post.It's about text retrieval ,I used to wonder how do search engines work exactly,sometimes they even seem smarter than an actual human .Search engines for example understand a lot of complex queries and also filter the results very accurately.
So there are a lot of resources on internet and based on some algorithms they retrieve the text which are useful for the user.
There are two algorithms for text retrieval ,one is document selection in which based on an algorithm's result the most relevant document is returned to the user ,the other is document ranking in which an rank is given to the documents by an algorithm based on the query of the user. The user then has the choice to pick the most important/relevant document needed .That's why document ranking is more preferred over document selection.
There are many algorithms for choosing the most appropriate documents based on the search query of the user.
Ranking of query results is the problem in text retrieval .Given a query Q and a collection of documents D that match the query the problem is to rank the documents in the most relevant order to the user.There are many ranking functions like : Pivoted Length Normalization, BM25, Query Likelihood.

Out of which BM25 is the most popular and would discuss it here.

BM25 (Best Matching 25)

BM25(Best Matching 25) is a bag of words retrieval function that ranks a set of documents based on a query terms appearing in each document.

What is TF* IDF ?

TF -> TF stands for Term frequency and it states that how often a term occurs in the document?
IDF -> Inversed Document Frequency .THe documents frequency measures how many docs a word appears in.
IDF (1/df) then measures how relevant the term is. Is the term very rare/uncommon? Or very common in the other documents.

If the word "football" is very common in a document but very rare in other documents its TF*IDF score will be very high.

BM25's on Term Frequency (TF Transformation)

c(w,d) in the ranking function shown above is directly multiplied but what if the word occurs too many times,so suppose if the word w occurs 500 times in a document d,we need to reduce that count as it would not lead to accurate ranking function otherwise.

Without any consideration for document length, term frequency follows the formula : 

((k + 1) * tf) / (k + tf)


More TF always means more relevance,however you never get past k, but you always approach it. The k value is often set between 0.9 - 1.2 ,if we set k to a higher value it will again act as a function shown above and create the same problem of TF.
