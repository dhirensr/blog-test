Honeypots in Apache HTTP Systems

After researching on Internet I found that it is very easily possible to log the passwords attempted wrongly by the user easily via the apache2.conf file. The apache2 conf file is a configuaration file for the apache service and settings of the services can be changed like where to log the error, how verbose the error logs should be and so on.
In the conf file on can write
LogFormat "{ \"ip\": \"%h\", \"time\": \"%t\", \"request\": \"%r\", \"status\": %>s, \"user_agent\": \"%{User-Agent}i\", \"authorization\": \"%{Authorization}i\" }" combined where it will now log the IP ,time ,request what the user made and also all the authorization details like what username and password did the user/hacker tried to use. The tricky part is that one would not understand easily that the password and username from authorization log is not obtained directly and after inspecting it I understood that it is in an encoded form. For decoding of the authorization one can use a2b base64 decoding and then one would obtain the username and password separated by ":".
